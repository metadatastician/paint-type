== What Is paint-type?

paint-type is a cross-platform open-source image editor in the spirit of Paint.NET —
capable enough for real work, simple enough to reach for without thinking.
License: PMPL-1.0-or-later. Owner: Joshua Jewell (JoshuaJewell on GitHub).

Part of the hyperpolymath/palimpsest ecosystem.

== Source Tree (`src/`)

=== `src/ephapax/` — Native Image Core (Rust)

`src/ephapax/` is paint-type's Rust crate (named "ephapax" for the role
it plays as the image core, not because it is the
`hyperpolymath/ephapax` language — those are SEPARATE things; the
language has sublanguages `ephapax-linear` + `ephapax-proven`, and this
crate is neither).

Modules:

`lib.rs`:: Tile API (`Tile::alloc/fill/read_pixel/write_pixel/drop`),
           f16↔f32 helpers, FFI declarations against libpt.
`composite.rs`:: Porter-Duff compositing operators (`over_premultiplied`,
           `over_unpremultiplied`, `masked_blend`, `flatten_layer_stack`,
           `Tile::composite_over`).
`undo.rs`:: Persistent branching undo graph (`UndoGraph<T>`, `RevId`)
           with the monotonicity invariant documented as a 4-clause
           contract for the eventual INV-2 Lean4 mechanisation.
`layer.rs`:: Layer model (`Layer`, `LayerStack`, `LayerId`) with stable
           IDs across reorderings and a `flatten(name)` builder.

Linear types enforce no aliased mutable tile access — `Tile` is `!Copy
+ !Clone`. Build with `cargo test` from `src/ephapax/`. Benches in
`src/ephapax/benches/`.

=== `src/interface/Abi/` — Formally Verified ABI Definitions (Idris2)

Three Idris2 modules prove correctness of the tile ABI:

`Types.idr`:: Tile primitive types — `TileHeader`, `TileBuffer`, `RGBA16F` pixel layout.
`Layout.idr`:: Memory layout proofs — `HasSize`, `HasAlignment`, layout correctness lemmas.
`Foreign.idr`:: FFI function wrappers — `pt_tile_alloc`, `pt_tile_free`, `pt_tile_blit`
             with dependent-type pre/post conditions.

=== `src/interface/ffi/` — Zig FFI Bridge

`src/interface/ffi/src/main.zig`:: Pure-Zig C-ABI implementation (`libpt`). Exports
  `pt_tile_alloc`, `pt_tile_free`, `pt_tile_fill`, `pt_tile_blit`,
  `pt_tile_read_pixel`, `pt_tile_write_pixel`, `pt_version`, and related
  functions via `callconv(.C)`. No `.c` or `.h` files in the repository —
  Rust links to the Zig-built shared library directly through `extern "C"`.
`src/interface/ffi/build.zig`:: Zig 0.15+ build file. Run `zig build test`.
`src/interface/ffi/test/integration_test.zig`:: Integration tests — lifecycle, blit
  operations, memory safety, version checks. Run with `zig build test`.

=== `src/bridges/` — AffineScript → typed-wasm Bridge (stub)

Future: generated from the Idris2 ABI definitions. Not yet populated.

=== `src/aspects/` — Cross-Cutting Concerns

Integrity, observability, and security cross-cuts. Scaffold in place; content planned for v0.2.0+.

== Verification (`verification/`)

Formal proofs and safety cases:

`verification/proofs/idris2/ABI/Platform.idr`:: Per-platform pointer + int size proofs (ABI-3). DONE.
`verification/proofs/idris2/ABI/Compliance.idr`:: C ABI compliance + tile-array packing (ABI-5). DONE.
`verification/proofs/idris2/Pixel.idr`:: RGBA16F IEEE-754 binary16 classifier + finiteness carriers (TP-3). DONE.
`verification/proofs/lean4/ApiTypes.lean`:: Lean4 type safety scaffold.
`verification/proofs/agda/Properties.agda`:: Agda inductive properties scaffold.
`verification/proofs/coq/TypeSafety.v`:: Coq type safety scaffold.
`verification/proofs/tlaplus/StateMachine.tla`:: TLA+ state machine scaffold.

== Session Protocol

`session/dispatch.sh`:: Maps canonical session commands to central protocol paths.
`session/custom-checks.k9`:: Repo-local K9 checks.
`session/local-hooks.sh`:: Local hook wiring.
`coordination.k9`:: Session coordination signals.

Central session protocol authority lives in `standards/session-management-standards/`.
Local files here are integration-only thin bindings.

Stapeln-based container definitions for running paint-type as a containerized service
(primarily for CI and headless render scenarios). Chainguard Wolfi base, signed with ML-DSA-87.

== CI/CD (`.github/workflows/`)

Key workflows:

`rust-ci.yml`:: Rust build and test for Ephapax.
`idris-ci.yml`:: Idris2 ABI proof type-check (`idris2 --check`) on ABI module + `verification/proofs/idris2/` changes.
`dogfood-gate.yml`:: A2ML, K9, eclexiaiser, Groove compliance.
`hypatia-scan.yml`:: Neurosymbolic CI/CD safety scan.
`static-analysis-gate.yml`:: Security static analysis.
`openssf-compliance.yml`:: OpenSSF Scorecard.
`codeql.yml`:: GitHub CodeQL.
`secret-scanner.yml`:: Secret detection.
`mirror.yml`:: Multi-forge sync.
