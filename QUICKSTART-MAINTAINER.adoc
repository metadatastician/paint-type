= paint-type — Quick Start for Platform Maintainers
:toc:
:toclevels: 2

== Overview

This guide covers building, packaging, and maintaining paint-type for distribution
on your platform.

== Runtime Dependencies

As of 2026-06-01, paint-type is a library/bridge crate (v0.2.0 closing — only
the AffineScript→typed-wasm bridge remains, gated upstream), not yet a
standalone binary. Runtime dependencies will be specified at v0.3.0 (Gossamer
shell integration).

For build dependencies, see: `guix.scm`.

== Build from Source

=== Prerequisites

* Rust stable toolchain
* Zig 0.15+
* Idris2 (for ABI proof verification)
* `just` task runner

=== Steps

[source,bash]
----
git clone https://github.com/metadatastician/paint-type.git
cd paint-type
just build-release
----

=== Zig FFI only

[source,bash]
----
cd src/interface/ffi
zig build -Doptimize=ReleaseSafe
----

=== Rust Ephapax only

[source,bash]
----
cd src/paint_core
cargo build --release
----

== Packaging

=== Guix

[source,bash]
----
guix build -f guix.scm
----

=== Container (Stapeln)

[source,bash]
----
just stapeln-export    # Generates signed container image
podman build -t paint-type .
----

Container is based on Chainguard Wolfi, signed with ML-DSA-87 via cerro-torre.

== Health Checks

[source,bash]
----
just doctor            # Full diagnostic
just test              # Run test suite
cd src/interface/ffi && zig build test   # Zig FFI tests
cd src/paint_core && cargo test             # Rust unit tests
----

== Updating

[source,bash]
----
git pull
just build-release
----
