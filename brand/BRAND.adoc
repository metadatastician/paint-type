= paint-type Brand Pack
:toc:

== Mark

The paint-type mark is a rose-hued brush stroke sweeping across a structured type glyph.
It represents the fusion of fluid artistic expression and rigorous formal types.

|===
| File | Use

| `icon.svg` | Primary mark with dark background (rounded corners). README, docs, about screens.
| `icon-square.svg` | Transparent background mark. Favicon, app icon, overlay on custom backgrounds.
| `social-preview.svg` | GitHub / Open Graph social preview (1280x640). "paint-type" wordmark + mark, no strapline.
|===

== Colours

[cols="1,1,2"]
|===
| Name | Hex | Use

| Rose Hue | `#f43f5e` | Primary brand colour. Brush stroke, active states, CTAs.
| Background | `#0d1117` | Dark surfaces. GitHub-dark compatible.
| Foreground | `#e6edf3` | Text on dark background.
| Glow | `#3f0a18` | Radial background glow behind mark.
|===

== Typography

Wordmark uses the system font stack: `system-ui, -apple-system, 'Segoe UI', Helvetica, Arial, sans-serif` at weight 300 (light).

No custom font is required. The wordmark is set in the SVG, not a separate font file.

== Rules

- The mark always includes the rose brush stroke intersecting the type structure.
- The strapline "Formal types for fluid paint." is used only in marketing materials, never in the product UI or repository assets.
- The rose hue must be `#f43f5e` — do not approximate with other pinks or reds.
- On light backgrounds, invert: dark crimson `#be123c` mark on white/light grey.

== Generating raster assets

[source,shell]
----
# Requires Inkscape or rsvg-convert (librsvg)

# Favicon (64x64 PNG)
rsvg-convert -w 64 -h 64 icon-square.svg > favicon-64.png

# App icon (512x512 PNG)
rsvg-convert -w 512 -h 512 icon.svg > icon-512.png

# Social preview (1280x640 PNG for GitHub)
rsvg-convert -w 1280 -h 640 social-preview.svg > social-preview.png
----
