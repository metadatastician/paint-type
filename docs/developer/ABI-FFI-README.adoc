= paint-type ABI/FFI Documentation
:toc: left
:source-highlighter: rouge

== Overview

paint-type follows the **Hyperpolymath RSR Standard** for ABI and FFI design:

* **ABI (Application Binary Interface)** defined in **Idris2** with formal proofs
* **FFI (Foreign Function Interface)** implemented in **Zig** for C compatibility
* **Native Rust consumer** wraps the FFI as a safe API in `src/ephapax/`
* **Any language** can call through the standard C ABI exposed by libpt

The native C symbol prefix is `pt_`; the shared library is `libpt.{so,a,dylib}`.

== Architecture

----
┌─────────────────────────────────────────────┐
│  ABI Definitions (Idris2)                   │

│  - Foreign.idr    (FFI declarations)        │

                  │
                  │ matches by name + signature
                  ▼
┌─────────────────────────────────────────────┐
│  FFI Implementation (Zig)                   │
│  src/interface/ffi/                         │
│  - src/main.zig         (pt_* exports)      │
│  - build.zig            (Zig 0.15.x build)  │
│  - test/integration_test.zig                │
└─────────────────┬───────────────────────────┘
                  │
                  │ compiled to libpt.{so,a}
                  ▼
┌─────────────────────────────────────────────┐
│  Native Rust consumer (`src/ephapax/`)      │
│  + Any other language via C ABI             │
└─────────────────────────────────────────────┘
----

== The libpt FFI Surface

As of v0.1.0, libpt exports these C-ABI functions (see
`src/interface/ffi/src/main.zig` for the canonical signatures):

[cols="1,3", options="header"]
|===
| Symbol | Purpose

| `pt_tile_alloc(x, y) -> u64`           | Allocate a 64×64 RGBA16F tile; returns opaque handle or 0 on OOM
| `pt_tile_free(tile_ptr)`               | Free a tile; no-op on null; double-free-safe via magic word
| `pt_tile_fill(t, r, g, b, a) -> u32`   | Fill every pixel with one f16 colour
| `pt_tile_read_pixel(t, px, py, &r, &g, &b, &a) -> u32` | Read one pixel
| `pt_tile_write_pixel(t, px, py, r, g, b, a) -> u32`   | Write one pixel
| `pt_alloc_u16_slot() -> u64`           | Allocate a single-u16 scratch slot
| `pt_read_u16_slot(slot) -> u16`        | Read scratch slot
| `pt_free_u16_slot(slot)`               | Free scratch slot
| `pt_last_error() -> char*`             | Static-buffer last-error message
| `pt_version() -> char*`                | Static-buffer semver string
| `pt_is_initialized(tile_ptr) -> u32`   | Sanity check: 1 iff libpt still recognises the tile
|===

== Why Idris2 for ABI?

=== 1. Formal Verification

Idris2's dependent types prove structural ABI properties at compile-time.
See `src/interface/Abi/Layout.idr` and `verification/proofs/idris2/ABI/`:

[source,idris]
----
-- The PtTile header is C-ABI compliant on every supported platform
public export
tileHeaderCompliant : CABICompliant ABI.Compliance.tileHeader
tileHeaderCompliant = ...

-- 4 * sizeof(uint32_t) = 16 on every supported platform
export
tileHeaderSize : (p : Platform) -> 4 * cInt32Size p = 16
----

=== 2. Type Safety at the Boundary

Encode invariants that C and Zig cannot express. See `src/interface/Abi/Types.idr`:

[source,idris]
----
-- Non-null pointer guaranteed at the type level
public export
record Handle where
  constructor MkHandle
  ptr : Bits64
  0 prf : So (ptr /= 0)
----

=== 3. Platform Abstraction

[source,idris]
----
-- Per-platform integer sizes (verification/proofs/idris2/ABI/Platform.idr)
public export
cInt32Size : Platform -> Nat
cInt32Size _ = 4  -- Proven equal across Linux64/LinuxARM64/MacOS64/Windows64/WASM32/...
----

== Why Zig for FFI?

=== 1. C ABI Compatibility

Zig exports C-compatible functions naturally:

[source,zig]
----
export fn pt_tile_alloc(x: u32, y: u32) u64 {
    const allocator = std.heap.c_allocator;
    // ...
}
----

=== 2. Memory Safety

Compile-time safety checks without runtime overhead.

=== 3. Cross-Compilation

Built-in cross-compilation to any platform:

[source,bash]
----
cd src/interface/ffi
zig build -Dtarget=x86_64-linux
zig build -Dtarget=aarch64-macos
zig build -Dtarget=x86_64-windows
----

== Usage Examples

=== From C

[source,c]
----
#include <stdint.h>

extern uint64_t pt_tile_alloc(uint32_t x, uint32_t y);
extern void     pt_tile_free(uint64_t tile_ptr);
extern uint32_t pt_tile_fill(uint64_t tile_ptr,
                             uint16_t r, uint16_t g, uint16_t b, uint16_t a);
extern const char *pt_version(void);

int main(void) {
    uint64_t tile = pt_tile_alloc(0, 0);
    if (!tile) return 1;
    pt_tile_fill(tile, 0x3C00, 0x0000, 0x0000, 0x3C00);  // opaque red
    pt_tile_free(tile);
    return 0;
}
----

Compile against the static lib:

[source,bash]
----
gcc example.c \
    -L./src/interface/ffi/zig-out/lib \
    -lpt -o example
----

=== From Idris2

[source,idris]
----
import Abi.Foreign
import Abi.Types

main : IO ()
main = do
  Just tile <- pure $ createTileHandle !(pt_tile_alloc 0 0)
    | Nothing => putStrLn "alloc failed"
  _ <- pt_tile_fill tile.ptr 0x3C00 0x0000 0x0000 0x3C00
  pt_tile_free tile.ptr
----

=== From Rust (via `src/ephapax/`)

[source,rust]
----
use ephapax::Tile;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let tile = Tile::alloc(0, 0).ok_or("alloc failed")?;
    tile.fill_f32(1.0, 0.0, 0.0, 1.0)?;          // opaque red
    let pixel = tile.read_pixel_f32(0, 0)?;
    assert_eq!(pixel, [1.0, 0.0, 0.0, 1.0]);
    // Tile drops here → pt_tile_free
    Ok(())
}
----

== Testing

* **Zig unit + integration**: `cd src/interface/ffi && zig build test`
* **Rust**: `cd src/ephapax && cargo test`
* **Idris2 proofs**: see "Idris2 ABI Type-Check" above; CI runs it via
  `.github/workflows/idris-ci.yml`.
* **Aspect tests**: `bash tests/aspect_tests.sh` — checks SPDX, banned
  patterns, ABI↔FFI contract (every Idris2 `%foreign "C:pt_X,libpt"`
  resolves to an `export fn pt_X` in Zig), Rust panic-safety, RGBA16F
  constants present in all three tiers.

== Contributing to the ABI/FFI

When modifying the ABI/FFI boundary:

. **Update the Idris2 ABI first** (`src/interface/Abi/*.idr`).
   * Modify type definitions
   * Update or add proofs
   * Keep backward compatibility where possible
. **Implement the Zig side** (`src/interface/ffi/src/main.zig`).
   * Match the Idris2 `%foreign "C:pt_X,libpt"` symbols exactly
   * Validate inputs; never trust the caller
. **Mirror the extern in Rust** (`src/ephapax/src/lib.rs`).
   * Add the `unsafe extern "C"` declaration
   * Expose a safe `Tile::*` method with `// SAFETY:` comments
. **Add tests**
   * Zig inline unit test in `main.zig`
   * Zig integration test in `test/integration_test.zig`
   * Rust unit test in `src/ephapax/src/lib.rs` or relevant module
. **Add aspect-test coverage** — Aspect 3 in `tests/aspect_tests.sh`
   verifies the ABI ⊆ FFI inclusion; if your new symbol breaks the
   pattern, fix the aspect test first.
